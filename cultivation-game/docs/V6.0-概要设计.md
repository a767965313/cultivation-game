## 二、核心模块设计

### 2.1 合成系统 (MergeSystem)

#### 算法流程
```
用户放置元素 → 触发检测 → BFS查找连通块
                                    ↓
                          连通数 ≥ 3 ?
                           /        \
                        是          否
                         ↓           ↓
                    执行合成      无操作
                         ↓
              计算纯度平均值
                         ↓
              检查完美合成条件
                         ↓
              生成新元素
                         ↓
              更新网格 + 播放动画
```

#### 核心函数
```typescript
interface MergeResult {
    newLevel: number;
    newElement: ElementType;
    isPerfect: boolean;
    bonusXP: number;
}

function checkMerge(grid: Grid, index: number): MergeResult | null
function findConnected(grid: Grid, start: number, element: ElementType): number[]
function executeMerge(indices: number[]): MergeResult
function calculatePurity(items: Item[]): number
function checkPerfectMerge(items: Item[], neighbors: Item[]): boolean
```

#### 完美合成判定
```javascript
// 条件1：所有参与合成的灵气纯度 ≥ 90%
// 条件2：至少有一个相生元素在相邻位置
// 满足任一即为完美合成，产出等级+1
```

## 三、状态管理

### 3.1 游戏状态 (GameState)
```typescript
interface GameState {
    // 基础
    slot: number;
    
    // 游戏区域
    grid: (Item | null)[];      // 64格合成器
    inventory: Item[];          // 库存
    selected: number | null;    // 选中的库存索引
    
    // 修仙进度
    realm: number;      // 0-8
    stage: number;      // 0-3
    xp: number;         // 当前修为
    
    // 统计
    capture: number;    // 捕捉次数
    merge: number;      // 合成次数
    stone: number;      // 吸收灵石数
    maxLevel: number;   // 最高灵石等级
    
    // 货币
    spiritStones: number;  // 灵石货币
    
    // 解锁
    shop: Record<string, number>;       // 已购买等级
    collection: Record<string, boolean>; // 图鉴解锁
    challenge: Record<string, { completed: boolean; claimed: boolean }>;
    pills: Record<string, number>;      // 丹药库存
    
    // 临时buff
    activeBuffs: Buff[];
}
```

## 四、交互设计

### 4.1 操作流程
```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  捕捉灵气  │ →  │ 选择放置  │ →  │ 触发合成  │ →  │ 收获灵石  │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
                                                        ↓
                              ┌──────────┐     ┌──────────┐
                              │  突破境界  │ ←  │  修炼吸收  │
                              └──────────┘     └──────────┘
```

### 4.2 手势/操作
| 操作 | 效果 |
|------|------|
| 点击「捕捉」 | 获得随机灵气（3-10个） |
| 点击库存 | 选中/取消选中 |
| 点击空格子 | 放置选中灵气 |
| 点击有物品的格子 | 取回物品到库存 |
| 点击「合成」 | 执行自动合成（优先最优解） |
| 点击「修炼」 | 吸收所有Lv1+灵石为修为 |
| 点击「更多」 | 打开功能抽屉 |
| 长按格子 | 查看物品详情（预留） |

### 4.3 反馈设计
| 事件 | 视觉反馈 | 触觉反馈 |
|------|----------|----------|
| 捕捉 | 图标弹出动画 | 轻震 |
| 放置 | 元素飞入 | - |
| 合成 | 闪光+粒子效果 | 轻震 |
| 突破 | 全屏渐变+文字 | 强震 |
| 解锁图鉴 | 卡片翻转动画 | - |
| 完成挑战 | 勋章弹出 | 轻震 |

```

4. 恢复正常界面 (200ms)
```

## 六、数值平衡

### 6.1 成长曲线
| 境界 | 修为需求 | 预计游戏时长 | 解锁内容 |
|------|----------|--------------|----------|
| 炼气期 | 100-400 | 5分钟 | 基础玩法 |
| 筑基期 | 200-800 | 15分钟 | 自动合成 |
| 金丹期 | 400-1600 | 1小时 | 扩展范围 |
| 元婴期+ | 指数增长 | 持续游玩 | 高级功能 |

### 6.2 经济平衡
- 初期：捕捉为主，合成效率低
- 中期：自动合成+商店升级
- 后期：高阶灵石+完美合成策略
